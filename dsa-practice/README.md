## The 7-Step Problem Solving Framework

Every problem follows this exact process — no shortcuts:

```
1. Read twice — understand input, output, constraints
2. Think out loud — what pattern does this look like?
3. Brute force — what is the naive O(n²) solution?
4. Optimize — which pattern reduces complexity?
5. Code — clean, readable, with type hints
6. Test — edge cases: empty, single element, negatives
7. Complexity — always state time AND space Big O
```

Never jump to coding. Steps 1-4 first, always.
